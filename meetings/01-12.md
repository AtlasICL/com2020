## Topics Discussed

### Project Selection
- Our team will be completing **Project 7**.

### Languages
- **Game:** Java, using the Google Style Guide.
- **Telemetry:** Python, using the PEP 8 Style Guide.

### Approach
- Our team will use an **Agile** approach.
- Two-week sprints will be used; sprint length is flexible depending on feature size.
- **Test-driven development (TDD)**.
- **Interfaces → Tests → Implementation**:
  - Identify all required interfaces.
  - Write all tests.
  - Implement features based on tests.

### File Structure & Security
- Two executables:
  - **Java (game)**
  - **Python (telemetry)**
- **Java application**:
  - User authentication.
  - Reads from a configuration file to set up the game.
  - Telemetry is written to the database but not displayed.
- **Python application**:
  - User authentication.
  - Standard players cannot access (RBAC).
  - Telemetry is read from the database and displayed on dashboards.
- JSON file for data storage must use authentication.
- Designers need to be able to change game rules in real time.

### Git & Jira
- Create a GitHub branch for each Jira ticket.
- **Do not commit source code to the `main` branch**.
  - Commits must be made only to feature-specific branches.
- CI/CD will be used on GitHub for automated testing.
  - **Emre** to configure this.
