----- NOTES -----

- We don't need to verify that the global point of access methods in Singleton classes are public. This is implicitly validated as the method can be successfully called from our Test class.
- Enums do not need unit testing, as they have no logic - they simply define named constants.
- Constructors only need testing when there is logic involved in the assignment of values.
- Any exception classes will be tested in the testing of any methods that throw it.
