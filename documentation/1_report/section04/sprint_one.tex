\subsection{Explanation}
Each subsection is an aspect of the project (either the documentation, Python, or Java application). Each subsubsection is then an epic within that area. Each paragraph represents one story for that epic, referencing a story in the project backlog. Each story is assigned a value associated with how much work is required to complete the story for this sprint, the risks associated with the story, the technical changes required to complete it, what it depends on, how complex it is to achieve, and what value it adds to the project, assuming all work to be done in the last sprint was done. Each story is also assigned a priority of either must, should, could, or wont. Lastly, an acceptance criteria is provided for each story, stating what tasks must be completed to achieve the story.

  \item The run continues until the simulated player beats the game or dies.

\subsubsection{Game Runs}

\paragraph{Users Can Choose Difficulty}
\textbf{Value:} 3\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item At the start of a game run, the player can select the difficulty.
  \item The difficulty options are Easy, Medium, and Hard.
  \item The game is harder on each subsequent difficulty (Easy is easiest; Hard is hardest).
  \item Difficulty affects gameplay by varying design parameter values.
\end{itemize}

\paragraph{Players Can Use Mundane Attack Abilities}
\textbf{Value:} 8\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item The player can always use the default attack ability on their turn during an encounter.
  \item The player cam use any purchased mundane attack abilities.
  \item These abilities deal the correct damage type as defined in the game design documentation.
  \item These abilities deal the correct amount of damage as defined in the game design documentation.
  \item The enemy's health is reduced by the calculated damage.
  \item All purchasable mundane attack abilities can be bought in the shop between encounters.
  \item If the player has less coins than the ability costs, it cannot be purchased.
  \item If purchased, the player is granted the ability, and its cost is deduced from their coins.
  \item They can be purchased at most once in a run.
\end{itemize}

\paragraph{Players Can Choose Which Abilities to Purchase}
\textbf{Value:} 1\quad \textbf{Priority:} Should\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Multiple abilities appear in the shop between encounters.
  \item Players can choose which ability they want to buy.
  \item Players are able to purchase multiple abilities from the same shop.
\end{itemize}

\paragraph{Players Can Use Purchased Magic Attack Abilities}
\textbf{Value:} 8\quad \textbf{Priority:} Could\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Players have a finite magic reserve limited by their maximum magic.
  \item Players can use purchased magic abilities if their magic reserve is greater than or equal to the ability's magic cost.
  \item Players cannot use magic abilities if they lack sufficient magic.
  \item Using a magic attack reduces their magic by its cost.
  \item These abilities deal the damage type specified in the game design documentation.
  \item These abilities deal the damage amount specified in the game design documentation.
  \item Enemy health is reduced by this amount.
  \item All purchasable magic attack abilities can be bought in the shop between encounters.
  \item If the player has less coins than the ability costs, it cannot be purchased.
  \item If purchased, the player is granted the ability, and its cost is deduced from their coins.
  \item They can be purchased at most once in a run.
\end{itemize}

\paragraph{Enemies Can Use Attack Abilities}
\textbf{Value:} 3\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Enemies are able to use both magic and mundane attack abilities.
  \item Enemies have attack abilities based on their type as defined in the game design documentation.
  \item These abilities deal the correct damage type.
  \item These abilities deal the correct damage amount.
  \item The player's health is reduced by this damage when attacked.
  \item Enemies with zero or less health are dead and cannot act.
\end{itemize}

\paragraph{Players Benefit from Purchased Damage Resistance Passive Abilities}
\textbf{Value:} 2\quad \textbf{Priority:} Could\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Players can purchase passive abilities granting resistance to specific damage types.
  \item When applicable, these abilities must halve incoming damage.
  \item If the player has less coins than the ability costs, it cannot be purchased.
  \item If purchased, the player is granted the ability, and its cost is deduced from their coins.
  \item They can be purchased at most once in a run.
\end{itemize}

\paragraph{Players Benefit from Purchased Damage Boost Passive Abilities}
\textbf{Value:} 2\quad \textbf{Priority:} Could\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Players can purchase passive abilities that boost specific damage types.
  \item When applicable, these abilities must double outgoing damage.
  \item If the player has less coins than the ability costs, it cannot be purchased.
  \item If purchased, the player is granted the ability, and its cost is deduced from their coins.
  \item They can be purchased at most once in a run.
\end{itemize}

\paragraph{Enemies Have Damage Resistances and Vulnerabilities}
\textbf{Value:} 2\quad \textbf{Priority:} Could\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Some enemies resist certain damage types.
  \item Some enemies are vulnerable to certain damage types.
  \item Incoming damage of a resisted type is halved.
  \item Incoming damage of a vulnerable type is doubled.
\end{itemize}

\paragraph{Players Can Encounter Multiple Enemies in Combat}
\textbf{Value:} 2\quad \textbf{Priority:} Could\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item Combat encounters support multiple enemies.
  \item Each enemy targets the player.
  \item The player can choose which enemy to target.
\end{itemize}

\paragraph{Players Can Complete a Combat Round}
\textbf{Value:} 1\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item At the start of each combat round, the player gains magic based on the game design documentation.
  \item The player then takes their turn.
  \item All enemies then take their turns.
  \item The round then restarts with magic regeneration.
\end{itemize}

\paragraph{Players Can Complete an Encounter}
\textbf{Value:} 1\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item An encounter is complete when all enemies have zero or less health.
  \item Upon completion, the player gains coins as defined in the game design documentation.
\end{itemize}

\paragraph{Players Can Retry an Encounter}
\textbf{Value:} 1\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item At the start of a run, the player has a number of lives defined by the design parameter.
  \item If the player's health reaches zero or less, they die.
  \item If the player has remaining lives, they restart the same encounter with reset health and magic.
  \item Enemy health must also be reset for the retry.
  \item The player loses one life upon death.
\end{itemize}

\paragraph{Players Can Progress Through Stages}
\textbf{Value:} 1\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item After completing an encounter, the player goes to the shop.
  \item After leaving the shop, the stage number increases.
  \item The player then enters another encounter.
  \item On stages 3, 6, 9, and 10, the player encounters a boss enemy.
  \item There are separate encounter pools for stages 1-2, 4-5, and 7-8.
  \item Encounters become progressively harder on higher stages.
\end{itemize}

\paragraph{Players Can Fail Runs}
\textbf{Value:} 1\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item If the player dies with zero remaining lives, the run ends.
  \item When this happens, a run-over screen is displayed.
  \item The screen shows the stage reached, run duration, coins collected, and deaths.
  \item The player loses all purchased abilities and reset to stage 1.
  \item The player can start a new run afterwards.
\end{itemize}

\paragraph{Players Can Complete Runs}
\textbf{Value:} 1\quad \textbf{Priority:} Must\\
\textbf{Acceptance criteria}
\begin{itemize}
  \item If the player completes the encounter on stage two, they complete the run.
  \item A run complete screen is displayed with stage reached, run duration, final coins, and deaths.
  \item The player can then start a new run.
\end{itemize}
