\section{Overview}
In this document we will outline first how to deploy both applications, and the test suite. We will then provide detailed operation instructions for both applications.

\section{Deployment}
\subsection{Prerequisites}
To be able to follow and complete deployment, as well as operate the applications, the following are required.
\begin{itemize}
  \item Internet access
  \item A web browser
\end{itemize}

Internet access is assumed, most supported operating systems contain a web browser, however if one is not installed, firefox can be installed by following the steps \href{https://www.firefox.com/en-US/download}{here}.

\subsection{From Executable}

\subsubsection{Build Process}
To compile the code into an executable form, first, assure you've replaced the placeholder strings in \verb|game\src\main\java\wizardquest\auth\Authenticator.java|\\ and \verb|telemetry\auth\auth.py| with their equivalent secrets, then run the following commands from the root of the \verb|com2020| directory:

\paragraph{Python Application}
\begin{verbatim}
cd telemetry
py -m PyInstaller --noconsole --onefile --name telemetry_app 
    --distpath . telemetry_app.py
\end{verbatim}

\paragraph{Java Application}
\begin{verbatim}
cd game
mvn clean compile -DskipTests
mvn javafx:jlink
\end{verbatim}

\subsubsection{Running the Applications}

\paragraph{Java Application}
\begin{verbatim}
cd game
target/image/bin/java -m wizard.quest/wizardquest.ui.GameRunPage
\end{verbatim}

\paragraph{Python Application}
\begin{verbatim}
cd telemetry
.\telemetry_app.exe
\end{verbatim}

\subsubsection{Installer Creation}
To create an installer for Windows, first, assure you're in the route of \texttt{com2020}, from the \texttt{Windows-Installer} branch. Then, execute the following using inno \\(more info at: https://jrsoftware.org/isdl.php/Inno-Setup-Downloads):

\begin{verbatim}
installer/WizardQuestInstall.iss
\end{verbatim}

The resulting installer will be generated at:

\begin{verbatim}
installer/output/WizardQuestInstaller.exe
\end{verbatim}


\subsection{From Source}
\subsubsection{Common}
These steps are required for both applications.

\begin{enumerate}
  \item Install python 3.13 or later. Instructions can be found \href{https://wiki.python.org/moin/BeginnersGuide/Download}{here}.
    \item Install requests: \verb|pip install requests|
  \item Setup environment variables. First obtain them from the email that is sent to all clients, then follow the below steps:
    \begin{itemize}
      \item On Windows run:
        \begin{enumerate}
            \item \verb|$env:OIDC_ISSUER="<ENVIRONMENT VARIABLE VALUE>"|
            \item \verb|$env:OIDC_CLIENT_ID="<ENVIRONMENT VARIABLE VALUE>"|
            \item \verb|$env:OIDC_CLIENT_SECRET="<ENVIRONMENT VARIABLE VALUE>"|
        \end{enumerate}
      \item On macOS run:
        \begin{enumerate}
            \item \verb|export OIDC_ISSUER="<ENVIRONMENT VARIABLE VALUE>"|
            \item \verb|export OIDC_CLIENT_ID="<ENVIRONMENT VARIABLE VALUE>"|
            \item \verb|export OIDC_CLIENT_SECRET="<ENVIRONMENT VARIABLE VALUE>"|
        \end{enumerate}
    \end{itemize}
\end{enumerate}

\subsubsection{Python Application}
\label{sec:python_src_deployment}
\begin{enumerate}
    \item Enter the telemetry directory: \verb|cd telemetry|
    \item Create a virtual environment: \verb|python3 -m venv venv|
  \item Activate the virtual environment:
    \begin{itemize}
        \item On windows: \verb|./venv/Scripts/Activate|
        \item On macOS: \verb|source venv/bin/activate|
    \end{itemize}
    \item Install packages: \verb|pip install -r requirements.txt|
  \item Run the program:
    \begin{itemize}
        \item On windows: \verb|python telemetry_app.py|
        \item On macOS: \verb|python3 telemetry_app.py|
    \end{itemize}
    \item To exit the venv run: \verb|deactivate|
\end{enumerate}

\subsubsection{Java Application}
\label{sec:java_src_deployment}
\begin{enumerate}
  \item Install java 17+. Instructions can be found \href{https://www.java.com/en/download/manual.jsp}{here}
  \item Install maven. Instructions can be found \href{https://maven.apache.org/install.html}{here}
    \item Enter the game directory \verb|cd game|
    \item Run \verb|mvn clean compile exec:java "-Dexec.mainClass=wizardquest.ui.GameRunPage"|
\end{enumerate}

\subsection{Test Suite}

\subsubsection{Python Application}
\begin{enumerate}
  \item Complete the deployment steps for the Python application in \ref{sec:python_src_deployment}
  \item While inside the virtual environment run: \verb|pytest|
\end{enumerate}
\subsubsection{Java Application}
\begin{enumerate}
  \item Complete the deployment steps for the Java application in \ref{sec:java_src_deployment}
  \item Run \verb|mvn test|
\end{enumerate}

\subsubsection{Main Menu}
Once authenticated, the main menu allows the user to start a new run of the game by pressing the "Start New Game" button. The "Run Simulation" button will execute three simulated runs, one on each difficult. It records the data from these runs in \\\verb|event_logs/simulation_events.json| which can be viewed in the telemetry app. The "Settings" button opens up the game settings and design parameter editor. The "Manage Roles" button opens the role assignment screen. The "Quit" button closes the game.

\centering

    \caption{Main menu screen}
\end{figure}

\subsubsection{Difficulty Select}
Once you start a new game, you will be prompted to select the difficulty of the game run. This screen displays four buttons, one for each difficulty, and a "back" button. Each difficult button states the difficulty, then in brackets how many lives you have on said difficulty, and lastly the farthest level you've reached on that difficulty. Selecting a difficulty will bring you to the first encounter of the run. Selecting "back" will take you back to the main menu. 

\begin{figure}[H]
\centering

\subsubsection{Encounter screen}
Once you start a run, and after each shop, you will enter an encounter. At the top of the screen it displays which stage you're on, and the encounter's name. There are three horizontal sections. Underneath those can appear a status message, informing you of why you cannot perform certain actions (for example trying to use a spell without sufficient magic), and Underneath that is a "Quit Run" button, that will take you back to the main menu. 

The left-most states your lives, and how many coins you have. Underneath this it states your health and then shows a bar representing how much health you have left, out of your maximum. Underneath that it states your magic, then how much magic you have out of your maximum. Underneath that it lists all attack abilities you've purchased throughout the run. 

In the middle section it will first show an ability select menu \ref{fig:java_encounter_screen}, where each button is the name of one of your purchased abilities, stating its damage and magic cost underneath the ability name. Once you select which ability to use by clicking its button, a target select menu will appear \ref{fig:java_target_select}. At the top this states what ability you are using, along with how much damage it deals, and how much magic it costs to use. Underneath this will be several buttons, where each button represents one enemy in the encounter, stating their name and how much health they have left, out of their maximum. Once clicked, the ability will be used on the enemy.

In the right-most section, each enemy is shown. The menu shows each enemy in the encounter, first stating their name, then how much health the have otu of their maximum, and underneath this showing their health bar.

\subsubsection{Shop}
In the shop menu, the title shows you how many coins you have in total. Underneath it shows each upgrade available in the shop, with each being their own button. It also shows that upgrade's cost in brackets next to it. At the bottom will be a status text box, stating when you lack coins to buy an upgrade, or how many coins you got from completing the previous encounter. Beneath that is a "leave shop" button which will take you to the next stage and its encounter.

\subsubsection{Run Complete}
Once you beat the encounter with the dragon, on stage ten, or if you run out of lives in an encounter, the game will show the run complete screen. This screen shows the furthest stage you reached that run, how many times you died during the run, and how many coins you finished the run with. Underneath this will be a "Main Menu" button, which when clicked returns you to the main menu.

\subsubsection{Settings}

In settings, the top section contains the design parameter values, with the difficulty labeled on the Y-axis and the design parameter's name on the X-axis. Below this is the it will display whether the user has telemetry enabled, and a button to toggle for the setting. When telemetry is enabled, the game will record telemetry events, and otherwise wont. Underneath this is a scrollable text box, containing a detailed telemetry disclosure, stating what data is collected and why. Underneath that is an editable text box, where designers can provide an optional justification for their design parameter value changes. Underneath that is the "save" button, which saves all design parameter changes, and records the justification in the design log. Lastly, the "back" button at the bottom of the screen takes the user back to the main menu.

\begin{figure}[H]
\centering

\subsubsection{Role Assignment}
In the role assignment screen, there will be a menu of each user in the system's ID, and next to them a role drop down. Beneath this it displays what user had their role updated most recently, and beneath that the "Refresh" and "Back" buttons. The drop down has the options of "DEVELOPER" "DESIGNER" and "PLAYER", and the drop down for your user ID will be greyed out. The refresh button re-reads the roles from the logins file, updating the roles in the menu. The back button takes you back to the main menu.

\subsubsection{Navigation}

The navigation bar contains eight options: "Home", "Funnel", "Difficulty Spike", "Health", "Coins", "Time", "Suggestions" and "Decision Log". Clicking on one of these will navigate to the corresponding page. By default, the home page will be open.

\subsubsection{Home}

From the home page, the user is greeted with a welcome message and three options.

The first option is a dropdown menu which allows users to pick from viewing the telemetry data gathered from players or the data gathered from simulated game runs.

The second option is to export gathered data to a .csv file. Either the player telemetry or simulated telemetry can be exported, depending on the selection in the dropdown menu.

The final option is to reset the data. Pressing this will open a confirmation dialog, and if the user confirms, all data corresponding to the dropdown selection will be deleted from the database.

\subsubsection{Telemetry Views}

At the top of the display, there are three checkboxes, which correspond to three different comparison modes: "Compare by Difficulty", "Compare by Time", and "Compare by Coin Hold". Selecting one of these will change the way data is presented within the telemetry views to allow for comparison between different levels of difficulty, time taken, or coins held. The user cannot select multiple comparison modes at the same time.
